# Module 3 – Security Frameworks and Controls

## Security Framework
- A security framework is a set of guidelines used to protect data and systems.
- It helps organizations manage risks and improve security.
- It also helps companies follow laws and regulations.

---

## Security Lifecycle
Security lifecycle is a continuous process.

Organizations:
- Identify risks
- Create policies
- Improve security
- Monitor systems regularly

---

## Purpose of Security Frameworks
- Protect personal information (PII)
- Secure financial data
- Find security weaknesses
- Manage organizational risks
- Support business goals

---

## PII (Personally Identifiable Information)

Examples:
- Name
- Contact number
- Email address
- Aadhaar number
- Home address

---

# Four Core Components of Security Frameworks

## 1. Identify and Document Security Goals
Organizations first decide their security goals.

Example:
- Following GDPR rules
- Protecting customer data

---

## 2. Set Guidelines to Achieve Goals
- Create policies and rules
- Rules for handling user data requests

---

## 3. Implement Strong Security Processes
- Apply security procedures and controls
- Verify users before updating account data

---

## 4. Monitor and Communicate Results
- Continuously monitor systems
- Report security issues to managers

---

# GDPR (General Data Protection Regulation)

- GDPR is a data protection law of the European Union.
- It gives users control over personal data.

---

# Security Controls

- Security controls are safeguards used to reduce security risks.
- They help protect systems and data.

Examples:
- Employee privacy training
- Password policies
- Antivirus software

---

# Role of Security Analyst

- Identify risks
- Monitor security alerts
- Implement security controls
- Track employee training
- Report security problems

---

# Difference Between Framework and Control

| Security Framework | Security Control |
|--------------------|-----------------|
| Overall security plan | Specific protection |
| Gives guidelines | Reduces risks |
| Big-picture approach | Practical actions |

---

# CIA Triad

CIA Triad is a basic security model.

CIA stands for:
1. Confidentiality
2. Integrity
3. Availability

---

# NIST Cybersecurity Framework (NIST CSF)

NIST = National Institute of Standards and Technology

- NIST CSF is a cybersecurity framework.
- It provides standards and best practices.
- It helps organizations manage cybersecurity risks.

## Purpose of NIST CSF
- Identify risks
- Protect assets
- Improve cybersecurity
- Reduce cyber attacks
- Support long-term security

---

# Threat Actor

A person or group that performs cyber attacks.

---

# Disgruntled Employees

- Unhappy employees can become threat actors.
- They may misuse sensitive information.

---

# Risk Reduction Methods

- Give limited access to employees
- Allow access only to required data
- Use monitoring and security controls

---

# Importance of Security Frameworks

- Help organizations manage risks
- Protect valuable assets
- Improve security policies
- Support security teams

---

# Security Control
Safeguards used to reduce security risks.

Examples:
- Firewall
- Antivirus
- Access control

---

## Security Framework
Guidelines used to manage security risks.

Four Components:
1. Identify goals
2. Set guidelines
3. Implement processes
4. Monitor results

---

## Compliance
Following laws and regulations related to security and privacy.

---

# Important Security Standards and Regulations

## NIST (National Institute of Standards and Technology)
- Provides cybersecurity standards and frameworks.
- Helps organizations manage cybersecurity risks.

---

## GDPR (General Data Protection Regulation)
- Protects user privacy and personal data.
- Used mainly in the European Union (EU).

---

## PCI DSS (Payment Card Industry Data Security Standard)
- Protects credit and debit card information.
- Used by organizations handling card payments.

---

## HIPAA (Health Insurance Portability and Accountability Act)
- Protects patient health information.
- Used in the healthcare sector.

---

## CIS (Center for Internet Security)
- Provides security controls and cybersecurity best practices.

---

## ISO (International Organization for Standardization)
- Develops international standards for security and technology.

---

## SOC 1 and SOC 2 (Systems and Organization Controls)
- Assess security, privacy, and access controls of organizations.

---

# Security Ethics

- Making responsible decisions in cybersecurity.
- Professionals must work honestly and legally.
- Main goal is to protect organizations and user data.

---

# Counterattacks

When an organization attacks back against a hacker.

---

# U.S. Standpoint on Counterattacks

- Counterattacks are generally illegal in the U.S.

Important Laws:
- Computer Fraud and Abuse Act (1986)
- Cybersecurity Information Sharing Act (2015)

---

# Risks of Counterattacks

- Can escalate attacks
- May create international conflicts
- Can damage innocent systems
- Difficult to control outcomes

---

# Important Point

Only authorized government or military personnel can conduct counterattacks.
