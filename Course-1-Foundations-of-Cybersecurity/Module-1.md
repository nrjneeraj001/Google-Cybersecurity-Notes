# Module 1 - Foundations of Cybersecurity

# Introduction
Cybersecurity is the practice of protecting systems, networks, devices, and data from cyber attacks and unauthorized access.

Organizations use security measures to reduce risks and respond to attacks.

---

# CIA Triad (Main Goals of Cybersecurity)

## Confidentiality
Only authorized users can access data.

## Integrity
Data should remain accurate and unchanged.

## Availability
Systems and data should be accessible when needed.

---

# Threat Actors
A threat actor is a person or group that creates security risks.

Examples:
- Hackers
- Cybercriminals
- Malicious insiders

---

# Types of Threats

## External Threats
Threats that come from outside the organization.

Example:
- Hackers

## Internal Threats
Threats that come from inside the organization.

Examples:
- Malicious employees
- Unauthorized data access

---

# Role of Security Teams

- Protect systems and data
- Reduce risks and damages
- Follow security regulations

- Protect customer trust

# Common Cybersecurity Terminology and Concepts

## Compliance
Following security standards and regulations.

## Security Framework
Guidelines for building security plans and reducing risks.

## Security Controls
Safeguards used to reduce security risks.

Examples:
- Firewall
- Antivirus

## Security Posture
An organization’s overall security strength.

Strong security posture = Lower risk

## Threat Actor
A person or group that creates security threats.

## Internal Threat
Threats caused by employees, vendors, or trusted users.

Types:
- Accidental
- Intentional

Examples:
- Clicking malicious links
- Unauthorized access

## Network Security
Protecting networks from attacks and unauthorized access.

## Cloud Security
Protecting cloud systems and cloud-based data.

## Programming in Cybersecurity
Programming is used to automate and improve security tasks.

Languages:
- Python
- SQL

---

# PII (Personally Identifiable Information)

Information that can identify a person.

Examples:
- Name
- Address
- Phone Number
- Email
- IP Address

---

# SPII (Sensitive PII)

Sensitive personal information requiring stronger protection.

Examples:
- Medical records
- Financial information
- Biometric data

---

# Identity Theft

Stealing personal information for fraud or financial gain.
